# 1. Linux and the terminal

Linux is a family of operating systems built around the Linux kernel. In laboratory work, “use Linux” usually means working in a Unix-like environment that provides:

- A **shell** (command interpreter)
- Standard command-line tools
- Files and directories
- Permissions
- Development tools

### Terminal prompt

Example prompt:

```bash
student@oslab:~$
```

How to read it:

- `student` → username
- `oslab` → machine name
- `~` → home directory
- `$` → normal user (root often uses `#`)

### Command structure

A typical Linux command has three parts:

- **command** (program name)
- **options** (flags that modify behavior)
- **arguments** (targets like paths)

Example:

```bash
ls -l /home/student
```

- `ls` → command
- `-l` → option
- `/home/student` → argument

### Case sensitivity

Linux is **case-sensitive**:

- `lab1`, `Lab1`, and `LAB1` are different names

# 2. Linux filesystem overview

Linux organizes storage as a single hierarchical tree that begins at the root directory: `/`.

Example top-level layout:

```text
/
|- bin
|- boot
|- dev
|- etc
|- home
|- lib
|- media
|- mnt
|- proc
|- root
|- run
|- sbin
|- tmp
|- usr
`- var
```

### Common directories (what they are for)

| Directory | Typical purpose |
|---|---|
| `/` | Root of the filesystem |
| `/home` | Personal directories for normal users |
| `/root` | Home directory of the root user |
| `/bin` | Basic user commands |
| `/sbin` | System administration commands |
| `/etc` | System-wide configuration files |
| `/dev` | Device files (hardware + virtual devices) |
| `/proc` | Kernel + process information (dynamic/virtual) |
| `/tmp` | Temporary files |
| `/usr` | Installed apps/libraries/docs |
| `/var` | Variable data (logs, cache, spool) |

### Example: exploring `/` and `/home`

```bash
cd /
ls
cd /home
ls
```

Example output:

```text
bin boot dev etc home lib media mnt proc root run sbin tmp usr var
student
```

> **Note about `/proc`:** many `/proc` files don’t exist on disk. They are generated by the kernel to show current system information.

# 3. Paths and navigation

A **path** tells Linux where a file or directory is located.

## Absolute vs relative paths

- **Absolute path**: starts from `/`
  - Example: `/home/student/lab1/notes.txt`
- **Relative path**: starts from your current directory
  - Example: `lab1/notes.txt`

## Special symbols

- `.` → current directory
- `..` → parent directory
- `~` → current user’s home directory

## `ls` — list directory contents


# 5. Permissions and executable files

When you run `ls -l`, the first field describes file type and permissions.

Example:

```bash
ls -l run.sh
```

Example output:

```text
-rw-r--r-- 1 student student 0 Mar 30 10:52 run.sh
```

Meaning of `-rw-r--r--`:

- First char `-` → regular file (`d` would mean directory)
- Next 3 `rw-` → owner permissions (read/write)
- Next 3 `r--` → group permissions (read)
- Next 3 `r--` → others permissions (read)

## `chmod` — change permissions

Add execute permission:

```bash
chmod +x run.sh
ls -l run.sh
```

Example output:

```text
-rwxr-xr-x 1 student student 0 Mar 30 10:52 run.sh
```

### Common error

```bash
./run.sh
```

Output:

```text
bash: ./run.sh: Permission denied
```

Fix it with `chmod +x`.

# 6. Pipes, redirection, and simple command combinations

Linux commands can be combined by redirecting output to files or piping output into other commands.

## Output redirection: `>` and `>>`

```bash
ls > filelist.txt
cat filelist.txt
echo "End of list" >> filelist.txt
cat filelist.txt
```

- `>` overwrites
- `>>` appends

## Input redirection: `<`

```bash
cat > numbers.txt << 'EOF'
3
1
2
EOF

sort < numbers.txt
```

Output:

```text
1
2
3
```

## Pipes: `|`

```bash
ls /etc | grep ssh
cat fruits.txt | grep banana
```

Example output:

```text
ssh
ssh_config
```

```text
banana
banana shake
```

# 7. Processes and basic job control

A **process** is a running program.

## `ps` — show processes

```bash
ps
```

Example output:

```text
PID TTY          TIME CMD
2154 pts/0    00:00:00 bash
2281 pts/0    00:00:00 ps
```

## `top` — live process view

```bash
top
```

`top` updates continuously. Press `q` to exit.

## `kill` — stop a process

```bash
kill 2289
```

## Background execution and jobs

Run in background:

```bash
sleep 60 &
```

Check jobs:

```bash
jobs
```

Bring job to foreground:

```bash
fg
```
